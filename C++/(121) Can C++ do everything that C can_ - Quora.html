If you're talking standard, portable C++, then the answer is no, there are several things C does that C++ cannot do.

C++ cannot declare same-type pointers to be non-aliasing. C does it with the keyword "restrict" (restrict type qualifier), C++ compilers usually have non-portable ways to do the same, since it's a very important and irreplaceable language feature.

Originally Answered: What are the things that C can do, but C++ cannot?

In terms of semantics, any language can do what another does, so lets talk only of syntax. It depends on what standard of C you compare against.

  • C89 - C++ is an almost 100% superset of this, so you can do everything
  • C99 - Has things like variable length arrays and flexible array members
  • C11 -Has type generic macros and array bounds checking

These syntactic constructs do not exist in C++, but you can achieve the same much more elegantly using C++ language constructs.
